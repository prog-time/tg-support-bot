# Руководство по работе с Admin Panel

> **Аудитория:** менеджеры поддержки, администраторы
> **Применимо:** только когда `MANAGER_INTERFACE=admin_panel` (см. `docs/switching-manager-interface.md`)

---

## Что такое Admin Panel

Веб-панель `/admin` — альтернативный интерфейс для команды поддержки. Менеджеры работают в браузере вместо Telegram-группы. Построена на Filament 3.

Новые сообщения от пользователей появляются автоматически — страница обновляется каждые **5 секунд** без перезагрузки.

---

## Доступ

Адрес: `https://<your-domain>/admin`

Для входа нужна учётная запись из таблицы `users`. Создать нового пользователя:

```bash
docker exec pet php artisan app:create-admin-user
```

---

## Разделы панели

### Диалоги (`/admin/conversations`)

Основной раздел для обработки обращений. Показывает список всех пользователей, написавших боту.

| Колонка | Описание |
|---|---|
| Chat ID | Идентификатор пользователя в его платформе |
| Платформа | `telegram`, `vk` или внешний источник |
| Последнее сообщение | Дата и время последнего сообщения |

**Фильтры:** по платформе (`telegram` / `vk`).

**Сортировка по умолчанию:** по дате обновления, новые сверху.

Нажмите на строку, чтобы открыть переписку.

---

### Страница диалога

Показывает историю переписки с конкретным пользователем.

- Входящие и исходящие сообщения отображаются в хронологическом порядке
- Страница обновляется каждые 5 секунд — новые сообщения появляются автоматически
- В режиме `admin_panel` — доступна форма ответа

**Отправка ответа:**
1. Введите текст в поле «Сообщение»
2. Нажмите кнопку отправки
3. Ответ сохраняется в БД и отправляется пользователю через очередь

> Форма ответа скрыта, если панель открыта в режиме `telegram_group`.

---

### Чаты (`/admin/bot-users`)

Список всех зарегистрированных пользователей бота с возможностью блокировки.

**Действия над пользователем:**

| Действие | Описание |
|---|---|
| Заблокировать | Устанавливает `is_banned = true`. Пользователь получит уведомление о блокировке вместо ответа |
| Разблокировать | Снимает блокировку (`is_banned = false`) |

---

### External Sources (`/admin/external-sources`)

> Доступно только администраторам (`role = admin`).

Управление внешними источниками, интегрированными через REST API.

**Создание источника:**
1. Нажмите «New External Source»
2. Укажите название и (опционально) `Webhook URL` — адрес, на который будут отправляться ответы
3. Сохраните — токен доступа генерируется автоматически

**Редактирование источника:**
- При сохранении **токен обновляется автоматически**
- Все интеграции, использующие старый токен, перестанут работать — обновите токен на стороне клиента

**Получение токена:**

```bash
docker exec pet php artisan tinker --execute="echo \App\Models\ExternalSource::find(1)->accessToken->token;"
```

---

### Пользователи (`/admin/users`)

> Доступно только администраторам (`role = admin`). Раздел находится в группе «Настройки».

Управление учётными записями для входа в `/admin`.

**Роли:**

| Роль | Что доступно |
|---|---|
| `manager` | Диалоги, Чаты |
| `admin` | Всё, включая Пользователи и External Sources |

**Создание пользователя через панель:**
1. Перейдите в «Настройки → Пользователи»
2. Нажмите «New Пользователь»
3. Заполните имя, email, пароль (минимум 8 символов) и роль

**Создание через artisan:**

```bash
docker exec pet php artisan app:create-admin-user
```

Команда запускает интерактивный диалог:

```
Creating a new admin panel user.

 Name: John Doe
 Email: john@example.com
 Password (min 8 characters):
 Confirm password:
 Role:
  [0] Администратор
  [1] Менеджер
 > 1

User "John Doe" (john@example.com) created with role "Менеджер".
```

Команда валидирует все поля перед сохранением: уникальность email, совпадение паролей, минимальную длину пароля. При ошибке выводит сообщение и завершается с кодом `1` без создания пользователя.

## Поведение в разных режимах

| Функция | `admin_panel` | `telegram_group` |
|---|---|---|
| Просмотр сообщений в `/admin` | Да | Да (read-only) |
| Форма ответа в `/admin` | Да | Скрыта |
| Создание форум-топиков в Telegram | Нет | Да |
| Новые сообщения сохраняются в БД | Да | Да |

## Частые вопросы

**Не вижу форму ответа.**
Проверьте, что `MANAGER_INTERFACE=admin_panel` в `.env` и контейнер перезапущен:
```bash
docker exec pet php artisan tinker --execute="echo config('app.manager_interface');"
```

**Новые сообщения не появляются.**
Страница обновляется каждые 5 секунд. Если сообщения не появляются после 10+ секунд — проверьте, работает ли очередь:
```bash
docker compose ps
```

**Пользователь заблокирован, но продолжает писать.**
При блокировке через панель отправка уведомления о блокировке происходит через очередь. Убедитесь, что worker запущен.
